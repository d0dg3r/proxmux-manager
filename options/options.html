Troubleshooting & Quick Start

SPICE Support

To use SPICE, your VM must have a Display set to "SPICE" in Proxmox. When you click the SPICE button, the extension downloads a .vv file. You need Virt-Viewer (remote-viewer) installed on your computer to open these files automatically.

SSH Shells

The Shell icon appears for Linux-based VMs and LXC containers when a valid IP address is detected. It uses the ssh:// protocol. Ensure your OS has a default handler for SSH links.

Pro Tip: For VMs, make sure the QEMU Guest Agent is enabled in Proxmox and installed in the VM, otherwise the extension cannot detect the IP address.

Self-Signed Certificates

If you use a self-signed certificate, the extension might fail to connect. Try opening your Proxmox URL in a new browser tab and click "Proceed to..." to trust the site manually.

Required Permissions

Your API Token needs VM.Console and Sys.Audit permissions. For SPICE, VM.Config.Options or PVEVMAdmin is required to fetch the configuration.

Firewall & Ports

Ensure your Proxmox node is reachable on port 8006. If you are behind a corporate proxy, you might need to whitelist the IP addresses of your cluster nodes.

Automatic Failover

Once you connect to your first node, the extension automatically discovers all other nodes in your cluster. If your primary node goes down, the extension will attempt to fetch data from the other nodes automatically.

Search and Filter Reset

The search box now includes a clear control and also supports quick reset with the Escape key. Search text, active filters, and expanded item state are restored when reopening the popup for a faster workflow.

Power Action Status Behavior

After start/stop/shutdown/reboot actions, the extension polls status endpoints and keeps a temporary confirmed-state override so list status remains accurate while cluster-wide resources synchronize.
